Cloud Adoption & Strategy

Developed a comprehensive cloud migration strategy for BallotOnline, an online voting company, evaluating AWS and Azure for cost efficiency, ease of deployment, and compliance.

Project 1: Overview of Cloud Technology

Developed an executive-level presentation advocating cloud migration for BallotOnline.

  • Explored cloud computing fundamentals by researching IaaS, PaaS, and SaaS models and their applications.
  • Identified strengths (scalability, cost savings), weaknesses (vendor lock-in, security concerns), opportunities (AI, automation), and threats (compliance challenges, cloud costs).
  • Evaluated financial implications with AWS TCO Calculator and Azure Pricing Calculator to compare costs and justify cloud migration.

Key Takeaways

Strengthened my ability to communicate technical concepts to non-technical audiences, developed financial impact assessment skills, and understood cloud adoption cost models.

Project 2: Cloud Risk and Compliance Issues Analysis

Assessed compliance and risk management strategies for BallotOnline’s cloud migration.

  • Identified key risks by evaluating security risks (misconfigurations, insecure APIs) and compliance concerns (GDPR, HIPAA, SOC 2).
  • Developed a compliance framework by implementing NIST Cybersecurity Framework principles, AWS IAM policies, and role-based access controls.
  • Created a risk management matrix through categorizing risks by likelihood and impact, recommending IAM, MFA, and security logging for mitigation.

Key Takeaways

Gained hands-on experience in cloud security frameworks, compliance mapping, and designing risk mitigation strategies tailored to enterprise cloud migration.

Project 3: Cloud Provider Evaluation & Cost Analysis

Researched and compared AWS and Azure cloud services for enterprise adoption, evaluating their cost efficiency and scalability.

  • Analyzed cloud networking architectures by comparing AWS VPC vs. Azure Virtual Network for secure networking and hybrid cloud setups.
  • Compared migration tools and cost structures through examining AWS SMS vs. Azure Migrate for workload transitions and cost optimization strategies.
  • Developed a recommended cloud strategy by proposing a hybrid cloud approach leveraging AWS for compute-intensive workloads and Azure for Microsoft 365 integrations.
  • Performed cost analysis by calculating Total Cost of Ownership (TCO) and Return on Investment (ROI) over a five-year period, determining that Azure provided a more cost-effective solution (~$4.06M vs. AWS’s ~$4.49M).

Key Takeaways

Refined my skills in multi-cloud architecture, cost-benefit analysis, and strategic decision-making in cloud adoption. Strengthened my ability to assess TCO, ROI, and financial considerations when migrating to the cloud.
